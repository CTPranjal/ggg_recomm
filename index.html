  • Priority 1 — Expiring Soon (<3 days): Surfaced first to prevent value loss. CleverTap fires "Voucher Expiry" push 48h before expiry.
  • Priority 2 — High Value: Vouchers with largest discount amount sorted next — increases perceived wallet value.

  • Filter: Excludes vouchers already applied or expired using event property "voucher_status = active".

👎 Explicit Feedback — Reinforcement Learning

  • Trigger: User taps "×" (Don't Like) or "Already Ate" button on any recommendation widget.
  • Immediate Action: Card hides instantly (opacity → 0, next item slides in). No friction for the user.
  • RL Signal sent to CleverTap: Event "recommendation_feedback" with properties: item_id, feedback_type (dislike/ate), recommendation_source, position_in_list.
  • Affinity Score Update: "Dislike" → lower brand_affinity score by -0.3. "Already Ate" → temporary exclusion for 7 days, then re-eligible.
  • Exclusion List: Item added to user's personal exclusion list — never shown again until explicitly cleared.
  • This is CleverTap Recommendations Roadmap item — Intellinode RL engine will process these signals to re-rank future recommendations.

📊 Implicit Feedback — Auto Re-ranking

  • Signal tracked: "recommendation_impression" event fires each time an item is displayed in the recommendation zone for that user.
  • Threshold rule: If impression_count ≥ 3 AND click_count = 0 → item flagged as "low relevance" for this user.
  • Action: CleverTap Intellinode automatically replaces flagged item with next-best candidate from ranking model.
  • Re-ranking signal: Impression-without-click → negative weight added to item_score for that user's profile. Score decreases: item moves down in the ranked list.
  • No user action needed: This is fully automatic — the system learns passively from non-engagement behavior.
  • Intellinode RL engine processes these signals batch-nightly initially, moving toward real-time scoring on the roadmap.
